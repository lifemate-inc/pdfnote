# Smart PDF モックアップ

Phase 0 で作成した HTML モックアップです。**実際にブラウザで開いて触れる**動くプロトタイプとして実装されています。

---

## 起動方法

`index.html` をブラウザで直接開くだけで動作します（サーバー不要）。

- Windows: `index.html` をダブルクリック、または Chrome / Edge にドラッグ
- VS Code から: Live Server 拡張機能で起動
- Claude Code: Launch Preview Panel で表示

---

## モックアップに含まれる画面

ヘッダー直下の青いナビバーから各画面に切り替えられます。

| 画面 | 内容 |
|------|------|
| **🏠 ホーム** | PDF ドロップエリア・「事前分割」モード案内・セキュリティ説明 |
| **📋 一覧** | サムネイルグリッド・左サイドバー（操作）・選択 / 抽出 / 回転 |
| **🖼 プレビュー** | 個別ページ表示・**鉛筆アイコンからのリネーム**・スタンプ・回転 |
| **✂️ 軽量化モード** | 「N ページずつに自動分割」する事前分割 UI |
| **抽出ダイアログ** | 範囲選択後の抽出フロー（ファイル名入力） |
| **不具合報告** | 個人情報を含まない報告フォーム |
| **重い PDF 警告** | 120 ページ超を検出した際の誘導モーダル |
| **ガイドツアー** | 初回起動時の 5 ステップ案内（ヘルプボタンから再表示可） |

---

## 確認できるインタラクション

- 鉛筆アイコンクリック → インラインリネーム → Enter / Esc で確定 or キャンセル
- サムネイルサイズの変更（左サイドバーのスライダー）
- 抽出ダイアログの開閉
- 画面遷移
- スタンプ配置の説明（実装本体は Phase 1）

---

## 制限事項（モックアップなので）

- **実際の PDF は読み込めません**（ダミーサムネイル表示のみ）
- **保存ボタンは動作しません**（実装は Phase 1）
- スタンプは見た目のみ（ドラッグ移動は未実装）
- ページ遷移ボタンは見た目のみ

これらは Phase 1 の MVP 開発で実装します。

---

## レビューしてほしいポイント

ユーザー（プロダクトオーナー）に確認していただきたいこと:

1. **画面構成は分かりやすいか？** — 介護事業所のスタッフが迷わず使えそうか
2. **ボタン・操作の名前は適切か？** — 「抽出」「事前分割」など業務用語として違和感ないか
3. **重要操作が目立っているか？** — 「PDFを開く」「抽出」「保存」など主要動線が見つけやすいか
4. **不足している機能はないか？** — モックアップに無い機能で必要なものがあれば指摘
5. **色・サイズ・余白は見やすいか？** — ノートPC画面で実際に使う想定でレビュー

フィードバックは口頭で OK です。「ここのボタンが分かりにくい」「ここに○○の機能が欲しい」など気軽にお伝えください。
