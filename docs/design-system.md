# PDFノート デザインシステム

**バージョン:** 0.1（Phase 0 ドラフト）
**最終更新:** 2026-05-23

このドキュメントは、PDFノート の UI を一貫した品質で実装するためのデザインガイドラインです。

---

## 1. デザイン原則

### 1.1 「IT素人でも 5 分で使える」
- 機能を意図的に絞る
- アイコンとテキストは必ず併記（アイコンのみ禁止）
- ボタンの目的が一目で分かる文言
- 重要操作と危険操作はビジュアル区別

### 1.2 「ノートPC画面で快適に使える」
- 主要操作はスクロールなしで完結
- 13-15 インチ画面で 15-20 ページ同時表示
- 横スクロールを発生させない

### 1.3 「安心して使える」
- 「ローカル動作中」を常時表示
- 個人情報を扱う UI で「PCから出ません」を明示
- 危険操作（削除等）は確認ダイアログ必須

## 3. タイポグラフィ

### 3.1 フォントファミリー
```css
font-family: 'Hiragino Sans', 'Yu Gothic', 'Meiryo', sans-serif;
```

Windows / Mac の両 OS で読みやすい日本語フォントを優先。

### 3.2 フォントサイズ
| 用途 | サイズ | Tailwind |
|------|--------|----------|
| ページタイトル | 24px | `text-2xl` |
| セクション見出し | 20px | `text-xl` |
| 本文（最小） | 16px | `text-base` |
| 補助テキスト | 14px | `text-sm` |

**ルール:** 本文の最小サイズは 16px。これより小さいテキストは原則使わない。

---

## 4. スペーシング

8px グリッドシステム。Tailwind のスペーシングをそのまま使用。

| 用途 | px | Tailwind |
|------|----|----|
| 要素内パディング | 8-16px | `p-2` / `p-4` |
| 要素間マージン | 16-32px | `m-4` / `m-8` |
| セクション間 | 32-48px | `my-8` / `my-12` |

## 5. コンポーネント

### 5.1 ボタン

**プライマリボタン（主要操作）**
```html
<button class="px-6 py-3 bg-blue-600 hover:bg-blue-700 text-white text-base font-medium rounded-lg shadow-sm transition">
  📥 PDF を開く
</button>
```

**セカンダリボタン（補助操作）**
```html
<button class="px-6 py-3 bg-white hover:bg-gray-50 text-gray-900 text-base font-medium rounded-lg border border-gray-300 transition">
  キャンセル
</button>
```

**危険ボタン（削除等）**
```html
<button class="px-6 py-3 bg-red-600 hover:bg-red-700 text-white text-base font-medium rounded-lg shadow-sm transition">
  🗑 削除する
</button>
```

**最小タッチターゲット:** すべてのボタンは最小 44×44px

### 5.2 アイコン
- 絵文字を活用（追加フォント不要・OS横断で安定表示）
- アイコンには必ず日本語ラベル併記
- 主要アイコン例:
  - 📥 開く / 📤 保存
  - ✏️ 編集 / 🗑 削除
  - 🔄 回転 / ↩ 戻る
  - ✂️ 分割 / 📑 抽出
  - 🏷 スタンプ
  - ❓ ヘルプ / 🐛 不具合報告

### 5.3 カード（サムネイル）
```html
<div class="bg-white border border-gray-200 rounded-lg p-3 hover:shadow-md transition cursor-pointer">
  <!-- サムネイル画像 -->
  <div class="aspect-[3/4] bg-gray-100 rounded mb-2"></div>
  <!-- ページ番号 -->
  <div class="text-sm text-gray-600 text-center">1 ページ目</div>
</div>
```

### 5.4 モーダル
- 背景: 半透明黒（`bg-black/50`）
- カード: 白・角丸・影
- 閉じるボタンは右上に必須

### 5.5 トースト通知
- 画面右上または右下に配置
- 3 秒で自動消去
- 成功・警告・エラーで色分け

## 6. レイアウト

### 6.1 グリッドシステム
- メインレイアウト: 12カラムグリッド
- サイドバー: 左 240px 固定
- メインコンテンツ: 残り全幅

### 6.2 ブレークポイント
| サイズ | 用途 | Tailwind |
|--------|------|----------|
| 1024px 〜 | ノートPC標準 | `lg:` |
| 1280px 〜 | デスクトップ | `xl:` |

> 小型画面（タブレット・スマホ）はスコープ外

---

## 7. アクセシビリティ

- WCAG 2.1 AA レベルを目標
- カラーコントラスト比 4.5:1 以上
- すべての操作はキーボードで完結可能
- フォーカスインジケーターを明示（`focus:ring-2 focus:ring-blue-500`）
- スクリーンリーダー対応の aria 属性
- 画像には alt 属性必須

---

## 8. 状態表現

### 8.1 ローディング
- スピナー + メッセージ
- 進捗バー（パーセンテージ表示）

### 8.2 空状態
- 大きめのアイコン + 説明文 + 主要操作ボタン

### 8.3 エラー
- 赤系アイコン + エラー内容（平易な日本語）+ 解決ヒント
- 「不具合報告」ボタンを併設

---

## 9. 文言ガイドライン

### 9.1 基本ルール
- 平易な日本語（小学生でも読める）
- カタカナ用語を避ける（例: 「ダウンロード」→「保存」）
- 命令形ではなく、丁寧な表現

### 9.2 例
| ❌ NG | ✅ OK |
|------|------|
| エラーが発生しました | このPDFは開けませんでした |
| インポート | 取り込み / 開く |
| エクスポート | 保存 / 書き出し |
| バリデーションエラー | 入力に間違いがあります |
| ファイルをアップロード | ファイルを選んでください |

---

## 10. 「ローカル動作中」表示

画面右下に常時表示する小さなバッジ:

```html
<div class="fixed bottom-4 right-4 bg-green-100 border border-green-300 text-green-800 px-3 py-2 rounded-full text-sm flex items-center gap-2">
  🔒 ローカル動作中
</div>
```

クリックで「PDFデータはお使いのパソコンから出ません」の説明モーダルが開く。
