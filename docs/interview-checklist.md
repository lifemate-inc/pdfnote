# パイロット協力事業所 ヒアリングチェックリスト

**用途:** Phase 0 Step 5（ユーザーヒアリング）で介護事業所を訪問する際に使用する質問リスト
**所要時間目安:** 30〜45 分
**最終更新:** 2026-05-23

---

## 訪問前の準備

- [ ] PDFノート モックアップ URL（または PC で開ける状態）を準備
- [ ] このチェックリストを印刷（または PC / タブレットで表示）
- [ ] サンプルPDF提供の同意書（任意・後述）
- [ ] 名刺・自己紹介資料

---

## ヒアリング項目

### 1. 基本情報

- [ ] 事業所種別（例: 訪問介護 / 通所介護 / 居宅介護支援 / 訪問看護）
- [ ] 規模（利用者数 / スタッフ数）
- [ ] 業務PC環境
  - [ ] OS（Windows 10/11 / macOS / その他）
  - [ ] メモリ（4GB / 8GB / 16GB / 不明）
  - [ ] 主に使うブラウザ（Chrome / Edge / Safari / Firefox / 不明）
- [ ] PCを使う人の構成（若手中心 / 中堅中心 / シニア中心 / 混合）
- [ ] ITサポートの有無（社内IT担当 / 外部委託 / なし）

---

### 2. 現状の大量PDF業務

**質問の前置き例:**
> 「毎月、ケアマネジャーさんなどから大量のPDFが送られてきて、それを人ごとに分割して各担当者に配るような作業があると伺いました。詳しく教えてください。」

- [ ] **どんなPDFがいつ届くか？**
  - 業務名（例: 「サービス利用票の月次配布」）
  - 発生頻度（毎月 / 隔月 / 不定期）
  - 1ファイルのページ数（おおよそ）
  - 1ファイルのサイズ（MB単位、分からなければ「重い」「軽い」程度）
  - PDFの構成（1人 ◯ ページ × 何人分か）
- [ ] **届いた後の作業フロー（現状）**
  - 受信方法（メール添付 / クラウド共有 / 物理メディア / その他）
  - 現状の作業ツール（Acrobat / 印刷 / その他）
  - 1ファイル処理にかかる時間（分単位）
  - 作業担当者（事務員 / 介護スタッフ / 管理者）
- [ ] **作業内容の具体化**
  - 分割する単位（人ごと / 期間ごと / 種類ごと）
  - リネーム規則（例: `氏名_月.pdf`）
  - 回転が必要なケース（横向き混在の頻度）
  - メモ書き・スタンプの需要（「確認済」「要対応」など）

---

### 3. 困りごと・改善希望

- [ ] **現状で困っていること**（自由回答）
  - 例: コスト、操作の難しさ、見づらさ、エラーなど
- [ ] **どこに最も時間がかかっているか？**
  - 分割 / リネーム / 確認 / 印刷 / その他
- [ ] **ミスが発生しやすいポイント**
  - 例: 間違った人にデータを渡してしまう、ページ抜けが起きる
- [ ] **既存ツール（Acrobat等）への不満**
  - 価格 / 機能過多 / UIの分かりにくさ / 重さ / その他
- [ ] **「こんな機能があったら…」という要望**（自由回答）

---

### 4. PDFノート への期待・受容性

**質問の前置き例:**
> 「今、こういうツールを開発しようとしているんです。モックアップを見ていただけますか？」

→ ここで `mockup/index.html` を見せる。各画面を順に説明・操作してもらう。

- [ ] **モックアップの第一印象**
  - 分かりやすそうか / 使えそうか / 戸惑いそうか
- [ ] **特に良いと感じた機能**
- [ ] **戸惑った操作 / 分かりにくかった画面**
  - 操作の説明なしで使えそうか確認
- [ ] **不足している機能**（自由回答）
- [ ] **「これなら使いたい」と思える？**
  - 「Acrobatから乗り換える価値があるか」も合わせて確認
- [ ] **無料・OSS であることへの反応**
  - 安心感 / 不安（サポートは？） / 価格より機能優先
- [ ] **「ローカル完結」「個人情報がPCから出ない」設計への反応**

### 5. パイロット運用への協力可否

- [ ] **Phase 1 完成後（2-3ヶ月後）にパイロット運用に協力してもらえるか？**
  - Yes / No / 条件付き Yes（条件: ）
- [ ] **パイロット運用の希望期間**（1ヶ月 / 業務サイクル1回分 / 継続）
- [ ] **フィードバック方法の希望**
  - 対面打ち合わせ / メール / 電話 / LINE / GitHub Issue
- [ ] **連絡頻度の希望**
  - 都度 / 週1 / 月1

---

### 6. サンプルPDF の提供（任意・最重要）

**質問の前置き例:**
> 「PoC（技術検証）のために、実際に使っている PDF のサンプルを提供していただけませんか？個人情報は完全に削除した上で提供してください。」

- [ ] **匿名化版PDF の提供可否**
  - Yes / No
- [ ] **提供方法**
  - USB / 暗号化メール / その他
- [ ] **匿名化作業の担当**
  - 事業所側で実施 / 開発側で立ち会い実施
- [ ] **データの利用範囲・廃棄時期**
  - 利用範囲: 「PDFノート の技術検証のみ」
  - 廃棄時期: 「Phase 0 終了時」（または合意した時期）

> **重要:** サンプルPDFは絶対にGitリポジトリにコミットしない。`.gitignore` で `*.pdf` を除外済み。
