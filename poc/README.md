# Smart PDF — Phase 0 技術検証 PoC

このディレクトリは Phase 0 Step 4（技術検証）用の PoC です。

## 検証目的

**120ページPDFがブラウザで実用速度で扱えるかを実証する。**

これがクリアできれば Phase 1 で PWA として実装。クリアできなければ「事前分割で軽量化する運用」をUI誘導に組み込むか、最悪 Electron 案に切り替え判断します。

---

## 起動方法

```powershell
cd projects/smart-pdf/poc
npm install
npm run dev
```

ブラウザが自動で開きます（`http://localhost:5173/`）。

---

## 検証手順

1. PoCを起動
2. **実際の120ページPDF**（介護現場で使う実データ・匿名化版）を読み込む
3. 自動で性能計測結果が表示される
4. 「分割テスト」ボタンで pdf-lib の分割性能を確認
5. 「File System Access API テスト」で保存先指定の動作確認
6. ブラウザを変えて（Chrome / Edge / Safari / Firefox）同じテストを繰り返す
7. PCを変えて（高スペック / 低スペック / Windows / Mac）同じテストを繰り返す

---

## 判定基準（Go / No-Go）

| 項目 | 基準 | 実測値 | 判定 |
|------|------|--------|------|
| 読み込み + サムネイル生成時間 | 30 秒以内 | 7.64 秒（144ページ・7.31MB） | ✅ |
| メモリ使用量（ピーク） | 2 GB 以内 | 23 MB | ✅ |
| 分割処理（5ページ） | 5 秒以内 | 67 ms | ✅ |
| File System Access API | Chrome / Edge で動作 | Chrome 148 で動作確認済み | ✅ |

3つともクリア → **ブラウザPWAで Phase 1 を進める判断**
1つでもNG → 「事前分割推奨フロー」を必須化するか、Electron 案を検討

> **✅ 2026-05-23 検証完了: 全項目クリア → ブラウザPWAで Phase 1 進行を決定**
